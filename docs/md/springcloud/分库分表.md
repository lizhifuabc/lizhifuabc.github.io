# 分库分表

分表

单表数据量太大，会极大影响 sql 执行的性能，单表到几百万的时候，性能就会相对差一些了，此时需要进行分表了。

分库

一个库一般经验而言，最多支撑到并发 2000，一定要扩容了，单库并发值最好保持在每秒 1000 左右。可以将一个库的数据拆分到多个库中。

## 拆分策略

**水平拆分**：将数据均匀放更多的库里，然后用多个库来扛更高的并发，还有就是用多个库的存储容量来进行扩容。

1. 多个库的多个表
2. 每个库的表结构都一样
3. 每个库表放的数据是不同的，所有库表的数据加起来就是全部数据。

**垂直拆分**：将较少的访问频率很高的字段放到一个表里去，然后将较多的访问频率很低的字段放到另外一个表里去

1. 多字段的表给拆分成多个表，或者是多个库上去
1. 每个库表的结构都不一样，每个库表都包含部分字段
1. 数据库是有缓存的，你访问频率高的行字段越少，就可以在缓存里缓存更多的行，性能就越好。

## 分库分表的方式

1. 按照数据范围进行拆分，比如按照时间范围
   - 扩容很简单
   - 容易产生热点问题，大量的流量都打在最新的数据上了。
2. 按照某个字段 hash 一下均匀分散
   - 可以平均分配每个库的数据量和请求压力
   - 扩容麻烦，会有一个数据迁移的过程
