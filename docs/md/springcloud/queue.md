# 消息队列

## 优点

- **解耦**

  通过一个 MQ，Pub/Sub 发布订阅消息这么一个模型，将消费者和生产者进行解耦。

- **异步**

  A 系统接收一个请求，需要在自己本地写库，还需要在 BCD 三个系统写库，此时通过异步的方式发送BCD，降低用户感知时间。

- **削峰**

  - 高峰期将消息积压在MQ中，系统根据最大处理请求进行处理消息。
  - 高峰期过了，再慢慢的进行消息消费，保障系统不会崩溃

## 问题

- 系统可用性降低，保障消息队列的高可用。
- 系统复杂度提高，保障消息重复消费，如何处理消息丢失。
- 一致性问题，通过异步操作，给用户的反馈速度是提高了，但是如何保障最终结果的一致性。
