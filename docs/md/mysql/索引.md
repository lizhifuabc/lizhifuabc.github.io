# 索引

```mysql
create database  `demo`;
drop table if exists `demo`.`demo`;
CREATE TABLE `demo`.`demo` (
`id` bigint(20) NOT NULL AUTO_INCREMENT COMMENT 'id',
`a1` varchar(255) NOT NULL COMMENT 'a1',
`a2` varchar(255) NOT NULL COMMENT 'a2',
`a3` varchar(255) NOT NULL COMMENT 'a3',
PRIMARY KEY (`id`),
index demo_idnex(a1,a2,a3)
) COMMENT='demo';
```



## 最左前缀原则

在MySQL建立联合索引时会遵守最左前缀匹配原则，即最左优先，在检索数据时从联合索引的最左边开始匹配。



例如创建复合索引：a1 a2 a3

单独使用：(使用索引)

```sql
explain select * from demo where a1 = '1221';
```

组合使用：

```sql
-- 走索引
explain select * from demo where a1 = '1221';
explain select * from demo where a1 = '1221' and a2 = '1';
explain select * from demo where a1 = '1221' and a3 = '1';
explain select * from demo where a1 = '1221' and a4 = '1';
-- MySQL有一个Optimizer（查询优化器），查询优化器会将SQL进行优化，选择最优的查询计划来执行。
explain select * from demo where a3 = '1221' and a1 = '1';
explain select * from demo where a2 = '1221' and a1 = '1';

-- 不走索引：根据最左匹配原则，没有 a1 的情况下，索引会失效
explain select * from demo where a2 = '1221' and a3 = '1';
explain select * from demo where a2 = '1221';
explain select * from demo where a3 = '1221';
explain select * from demo where a3 = '1221' and a2 = '1';
```



如上测试，可以看到只要查询条件的列中包含组合索引最左边的那一列，不管该列在查询条件中的位置，都会使用索引进行查询。
